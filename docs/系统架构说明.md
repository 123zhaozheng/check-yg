# 员工-客户金额往来审计系统 - 架构说明

> 这是一份通俗易懂的系统说明文档，帮助你理解这个 GUI 应用的代码结构、数据流转和业务逻辑。

---

## 一、这个系统是干什么的？

简单说：**帮银行/企业检查员工有没有和特定客户发生金钱往来**。

想象一下这个场景：
- 银行有一批员工的银行流水（PDF/Excel 文件）
- 银行有一份"需要关注的客户名单"
- 需要检查：员工的流水里有没有和这些客户的交易记录

手动一条条查？太累了！这个系统就是自动化完成这个工作的。

## 二、系统的四个主要步骤（页面）

整个系统就像一条流水线，分四步走：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 流水提取  │ → │  2. 流水预览  │ → │  3. 审查配置  │ → │  4. 结果展示  │
│  ExtractPage │    │  PreviewPage │    │  ReviewPage  │    │  ResultPage  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### 步骤1：流水提取（ExtractPage）
**做什么**：从一堆 PDF/Excel 文件里，把银行流水数据"挖"出来

**输入**：一个文件夹，里面放着各种银行流水文件
**输出**：结构化的流水数据（谁给谁转了多少钱、什么时候转的）

**怎么做的**：
1. 扫描文件夹，找到所有 PDF/Excel 文件
2. 用 AI（大模型）分析每个文件里的表格
3. AI 判断哪些表格是"银行流水表"
4. 从流水表里提取关键信息：交易时间、对方姓名、金额等

### 步骤2：流水预览（PreviewPage）
**做什么**：让你看看提取出来的数据对不对

**功能**：
- 显示提取了多少条流水
- 可以搜索、排序
- 可以导出成 Excel 文件
- 确认没问题后，点"开始审查"进入下一步

### 步骤3：审查配置（ReviewPage）
**做什么**：上传客户名单，配置匹配规则

**输入**：
- 流水 Excel（上一步自动传过来）
- 客户名单 Excel（你手动选择）

**配置项**：
- 精确匹配：名字必须完全一样（张三 = 张三）
- 脱敏匹配：支持带星号的名字（张* 能匹配 张三）

### 步骤4：结果展示（ResultPage）
**做什么**：显示审查结果

**展示内容**：
- 命中了多少客户
- 有多少条匹配记录
- 涉及多少金额
- 风险等级（低/中/高）
- 详细的匹配明细表

## 五、核心类说明

### 1. FlowRecord（流水记录）
一条银行流水的数据结构：

```python
@dataclass
class FlowRecord:
    source_file: str          # 来源文件名
    original_row: int         # 原始行号
    transaction_time: str     # 交易时间
    counterparty_name: str    # 交易对手姓名 ← 这是匹配的关键字段！
    counterparty_account: str # 交易对手账号
    amount: str               # 金额
    summary: str              # 摘要/备注
    transaction_type: str     # 收支类型（收入/支出）
```

### 2. NameMatcher（姓名匹配器）
负责判断两个名字是否"匹配"：

```python
# 精确匹配：名字完全一样
"张三" == "张三"  ✓

# 脱敏匹配：支持星号通配
"张*" 匹配 "张三"  ✓
"*三" 匹配 "张三"  ✓
"张*辰" 匹配 "张北辰"  ✓

# 模糊匹配：相似度计算（可选）
"张三" 和 "张叁" 相似度 80%  ✓
```

### 3. FlowExtractor（流水提取器）
整个提取流程的"总指挥"：

```python
class FlowExtractor:
    def extract_flows(self, document_folder):
        # 1. 扫描文件夹
        documents = self.scanner.scan_directory(document_folder)
        
        # 2. 逐个处理文件
        for doc in documents:
            # 2.1 解析文件，提取原始表格
            raw_tables = self._extract_raw_tables(doc)
            
            # 2.2 AI 分析每个表格
            for table in raw_tables:
                mapping = self.table_analyzer.analyze_table(table)
                
                # 2.3 如果是流水表，提取记录
                if mapping.is_flow_table:
                    records = self.table_analyzer.extract_rows(table, mapping)
                    all_records.extend(records)
        
        return ExtractionResult(flow_records=all_records)
```

## 六、页面之间的信号传递

PyQt5 使用"信号-槽"机制来实现页面间通信：

```python
# MainWindow 中连接信号
class MainWindow:
    def _connect_signals(self):
        # 提取完成 → 跳转到预览页
        self.extract_page.extraction_completed.connect(self._on_extraction_complete)
        
        # 预览页点击"开始审查" → 跳转到审查页
        self.preview_page.audit_requested.connect(self._on_audit_requested)
        
        # 审查开始 → 执行审查并跳转到结果页
        self.review_page.start_review.connect(self._on_review_start)
```

信号流转示意：

```
ExtractPage                PreviewPage               ReviewPage                ResultPage
    │                          │                         │                         │
    │ extraction_completed     │                         │                         │
    ├─────────────────────────→│                         │                         │
    │                          │ audit_requested         │                         │
    │                          ├────────────────────────→│                         │
    │                          │                         │ start_review            │
    │                          │                         ├────────────────────────→│
    │                          │                         │                         │
```

## 七、配置管理

所有配置存储在 `~/.check-yg/config.yaml`：

```yaml
mineru:                    # PDF 解析服务配置
  url: http://localhost:8000
  timeout: 300

llm:                       # AI 大模型配置
  url: https://api.openai.com/v1
  model: gpt-4
  api_key: sk-xxx
  batch_size: 10           # 并发请求数
  match_threshold: 70      # 匹配置信度阈值

matching:                  # 匹配选项
  enable_exact: true       # 启用精确匹配
  enable_desensitized: true # 启用脱敏匹配
  enable_fuzzy: false      # 启用模糊匹配

paths:                     # 路径配置
  input_folder: ./data/input
  output_folder: ./data/output
  reports_folder: ./data/reports
```

## 八、业务逻辑总结

### 核心业务流程

```
┌──────────────────────────────────────────────────────────────────────────┐
│                           员工-客户金额往来审计                            │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  输入：                                                                   │
│  ├── 员工银行流水文件（PDF/Excel）                                         │
│  └── 需关注的客户名单（Excel）                                             │
│                                                                          │
│  处理：                                                                   │
│  ├── 1. 从流水文件中提取交易记录                                           │
│  ├── 2. 对每条记录的"交易对手"字段                                         │
│  └── 3. 与客户名单进行匹配（精确/脱敏/模糊）                                │
│                                                                          │
│  输出：                                                                   │
│  ├── 匹配到的交易记录列表                                                  │
│  ├── 涉及的客户数量和金额                                                  │
│  └── 风险等级评估                                                         │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```

### 为什么需要 AI？

银行流水文件格式五花八门：
- 不同银行的 PDF 格式不同
- 表头名称不统一（"交易对方"/"对方户名"/"收款人"）
- 有些是扫描件，需要 OCR

AI 的作用：
1. **判断表格类型**：这个表格是不是银行流水？（置信度评分）
2. **识别字段映射**：哪一列是"交易时间"？哪一列是"金额"？
3. **提取结构化数据**：把杂乱的表格变成标准格式

## 十、常见问题

### Q: 为什么提取很慢？
A: 主要是 AI 分析需要时间。每个表格都要调用大模型 API，网络延迟 + 模型推理时间。

### Q: 脱敏匹配是什么意思？
A: 银行流水里的名字经常被打码，比如"张*"、"*三"。脱敏匹配就是能把这些打码的名字和完整名字对应上。

### Q: 为什么有时候匹配不准？
A: 可能原因：
- 名字太短（2个字）容易误匹配
- OCR 识别错误（"明"识别成"眀"）
- 流水里是昵称/别名

### Q: 配置文件在哪里？
A: Windows: `C:\Users\你的用户名\.check-yg\config.yaml`
